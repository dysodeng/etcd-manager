# 登录页与集群概览视觉优化设计

## 背景

当前新版 Web 界面整体方向已确定，但实际截图暴露出两处视觉问题：登录页左侧说明文案因可用宽度不足产生换行，表单外置标签显得松散；集群概览的集群 ID 与 Leader 使用整行描述条，占用高度且信息层级不清晰。

## 目标

- 登录页左侧产品说明在当前桌面布局中保持单行，视觉重心更稳定。
- 登录表单去除外置字段标签，通过图标、占位文案和无障碍属性表达字段含义。
- 集群成员卡片将集群 ID 与 Leader 收拢为紧凑摘要，不再占用一整条空白区域。
- 保持既有登录、集群数据请求、权限、主题与响应式边界不变。

## 设计

### 登录页

- 保留左右分栏结构、品牌标识、主标题、功能标签与安全提示。
- 左侧说明文案扩大最大宽度，并在目标桌面宽度下禁止换行；必要时略微降低字号，避免与右侧面板竞争空间。
- 删除 `Form.Item` 的可见 `label`，保留必填校验规则。
- 用户名与密码输入框继续使用前置图标和明确的 placeholder，并增加 `aria-label`，确保去掉可见标签后仍可被辅助技术识别。
- 适当缩短字段之间的垂直间距，让表单成为一个连续、紧凑的登录区块。

### 集群概览

- 删除成员卡片正文顶部的 `Descriptions` 横条。
- 在“集群成员”卡片标题区右侧展示两个紧凑摘要项：集群 ID 与当前 Leader。
- 摘要项使用弱背景、细边框与等宽/强调文本区分标签和值；Leader 使用信息色强调，但不与健康状态的绿色混淆。
- 表格直接接在卡片标题区之后，减少无意义留白。
- 保留成员表格、Leader/Voter/Learner 状态表达、数据字段和请求逻辑不变。

## 测试与验收

- 先补充组件结构测试并确认旧实现失败：登录表单不再输出可见 label，输入框具备无障碍名称；集群成员卡片不再渲染 `Descriptions`，而是渲染紧凑摘要。
- 完成实现后运行相关测试、全量 Web 测试、lint、typecheck 与生产构建。
- 以用户提供的截图为视觉基准：登录说明保持单行，表单不再出现突兀标签；集群成员卡片中不再出现整行空白摘要条。

## 非目标

- 不修改登录接口、认证流程或默认账号。
- 不改变集群指标、表格列、请求频率或错误处理。
- 不重新设计其他业务页面。
