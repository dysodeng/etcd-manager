{{- if .Values.etcd.enabled }}

A bundled {{ .Values.etcd.replicaCount }}-node etcd cluster (StatefulSet
"{{ include "etcd-manager.etcd.fullname" . }}") was created. The backend is
pre-configured to connect to 127.0.0.1 by default, so point it at the cluster:

  ETCD_ENDPOINTS={{ include "etcd-manager.etcd.fullname" . }}-client.{{ .Release.Namespace }}.svc:{{ .Values.etcd.service.clientPort }}

e.g. override backend.env in your values:

  backend:
    env:
      ETCD_ENDPOINTS: "{{ include "etcd-manager.etcd.fullname" . }}-client.{{ .Release.Namespace }}.svc:{{ .Values.etcd.service.clientPort }}"
{{- end }}

1. Wait for the Pods to become ready:

   kubectl --namespace {{ .Release.Namespace }} rollout status deploy/{{ include "etcd-manager.backend.fullname" . }}
   kubectl --namespace {{ .Release.Namespace }} rollout status deploy/{{ include "etcd-manager.frontend.fullname" . }}

2. Access the UI.{{- if .Values.ingress.enabled }}
   An Ingress is enabled for host(s):
{{- range .Values.ingress.hosts }}
     - http{{ if $.Values.ingress.tls }}s{{ end }}://{{ .host }}{{- end }}
{{- else }}
   Port-forward the frontend Service:

     kubectl --namespace {{ .Release.Namespace }} port-forward svc/{{ include "etcd-manager.frontend.fullname" . }} 8080:{{ .Values.frontend.service.port }}

   Then open:  http://127.0.0.1:8080

   (The frontend calls /api/v1, which Ingress routes to the backend. With a
   plain port-forward use the Ingress or a combined port-forward instead.)
{{- end }}

3. SECURITY: change the JWT secret and the admin password before going to
   production. Supply a strong random value for the JWT secret, e.g. via a
   Secret + envFrom:

     backend:
       envFrom:
         - secretRef:
             name: etcd-manager-secrets   # must define JWT_SECRET

{{- if .Values.etcd.enabled }}
NOTE on the bundled etcd cluster: a StatefulSet's volumeClaimTemplates are
IMMUTABLE in Kubernetes. If you ever change etcd persistence (size, storageClass)
or see "updates to statefulset spec ... are forbidden" on helm upgrade, delete
the StatefulSet first (this preserves the PVCs/data) then re-run the upgrade:

   kubectl --namespace {{ .Release.Namespace }} delete statefulset {{ include "etcd-manager.etcd.fullname" . }} --cascade=orphan
   helm upgrade etcd-manager deploy/helm -n {{ .Release.Namespace }}
{{- end }}
